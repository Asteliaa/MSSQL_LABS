Задание 4
Управление безопасностью
Создайте для выполнения заданий базу данных Test
    1. Режим проверки подлинности и настройка имени входа:
        1.1. Измените режим проверки подлинности сервера на смешанный (если до этого был режим Windows Authentification);
        1.2. Создайте имя  входа TestLogin1, укажите для него пароль (выберите аутентификацию SQL Server Authentification);
        1.3. Добавьте созданное имя входа (или регистрационную запись) в фиксированную серверную роль sysadmin;
        1.4. Установите в качестве используемой по умолчанию БД для имени входа TestLogin1 базу данных Test;
        1.5. Создайте имя входа TestLogin2, задайте для него пароль. Создайте пользователей БД Test с именами TestUser1, TestUser2. Пользователь TestUser1 должен соответствовать имени входа TestLogin1, а  TestUser2 – TestLogin2.
    2. Настройка пользовательских ролей:
        2.1. Создайте две пользовательские роли Manager и Employee в БД Test. 
        2.2. Задайте для  пользователя TestUser1 роль Manager, а для пользователя TestUser2 – роль Employee. Запретите для роли Employee изменение пользователя guest. 
    3. Создайте новую Роль в БД и запретите ей обновлять таблицы в БД.
    4. Для выполнения этого задания дополнительно используйте материал, представленный в Topic 2.
4.1 Создайте любую таблицу в новой схеме БД Test (Transact - SQL), принадлежащей пользователю TestUser1.
4.2 Создайте в БД Test двух пользователей User1, User2. Добавьте их в пользовательскую роль Manager (п.2). Реализуйте несколькими способами следующую задачу: запретите User1, User2 выборку данных из таблицы, созданной в п.4.1 (пример с T-SQL обязателен).
